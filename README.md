# AIAnalyzer

Local-first analyzer for AI-coding sessions. Ingests sessions from GitHub Copilot CLI (Claude Code, OpenAI Codex CLI, and VS Code Copilot Chat are planned), computes usage signals, and classifies your collaboration archetype.

> Status: **M0–M4 (Portal)**. Copilot CLI only. See `DESIGN.md` for the full roadmap.

## Install

```bash
pip install -e .
```

Requires Python 3.11+.

## Usage

### Terminal report

```bash
# 1. Ingest all local Copilot CLI sessions into a DuckDB cache.
aianalyzer scan

# 2. Aggregate and print your archetype report.
aianalyzer report
```

### Web portal

```bash
# Launch the local portal at http://127.0.0.1:8765/
aianalyzer serve
```

The portal renders:

- **Hero card** — your primary archetype, secondary tilt, modifier tags, and planning/control axis scores with a confidence band.
- **12 KPI cards** — total sessions, turns, hours, days active, longest streak, averages for turns/minutes/prompt words (mean + median + p90), acceptance rate, first session timestamp.
- **5 charts** (Chart.js, vendored locally) — session-purpose doughnut, top-tools bar, 90-day activity line, hour-of-day bar, weekday bar.
- **Top-N tables** — projects, models, file extensions.
- **AI profile narrative** — a Markdown report generated by your local `copilot` CLI, summarising your collaboration style in 4 sections (who you are, how you work, strengths, suggestions).

Portal flags:

- `--host 0.0.0.0` to expose on the LAN (default `127.0.0.1`).
- `--port 9000` to pick a different port (default `8765`).
- `--no-open-browser` for headless / SSH use.

Click **Scan sessions** in the top bar to re-ingest after new Copilot CLI activity; the profile refreshes when the job completes. Click **Generate AI profile narrative** to invoke the local `copilot` binary — this typically takes 1–3 minutes.

## Archetypes

Two axes — **planning** (how much you think before prompting) × **control** (how much you steer the AI hands-on) — yield four primary archetypes:

| Planning | Control | Archetype | Sketch |
| --- | --- | --- | --- |
| High | High | **Architect** | Designs first, drives tools |
| High | Low  | **Pilot**     | Plans, then lets the AI fly |
| Low  | High | **Tinkerer**  | Hands-on, exploratory |
| Low  | Low  | **Vibe Coder**| "Just build it, ship it" |

Modifier tags (`questioner`, `debugger`, `planner`, `yolo`, `parallelist`) add nuance on top. Weights live in `src/aianalyzer/classifier/weights.yaml` — tune them for your team.

The portal additionally classifies each session into one of 10 **session purposes** (feature work, debugging, testing, refactoring, code review, learning/exploration, documentation, configuration, planning, casual chat).

## Privacy

Everything stays on disk. No network calls except the `copilot` CLI subprocess that the portal invokes for the narrative (and that subprocess obeys whatever auth / network policies your local `copilot` install already uses). PII redaction (PATs, Bearer tokens, AWS keys, `password=` strings, emails) runs at the normalization stage. The portal binds to `127.0.0.1` by default.
